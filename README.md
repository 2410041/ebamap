# EBAMAP 開発ルール

## 使用技術
- フロントエンドは React を使用する
- バックエンドは PHP を使用する
- データベースは MySQL / SQL を使用する
- PWA対応する
- Node.js / Express / FastAPI / Java は使用しない

## アプリの目的
EBAMAPは、スーパー店内でお客様や店員が商品を検索し、商品の場所・ポイント対象商品・お気に入り・検索履歴を確認できるPWAである。

このアプリの核は「商品を検索すると、どこにあるか分かる」ことである。

## 優先機能
1. 店舗選択
2. 商品検索
3. 商品位置表示
4. 店内マップ表示
5. ポイント対象商品表示
6. お気に入り
7. 検索履歴
8. PWA対応

## リアルタイム更新
- WebSocketは使わない
- ReactからPHP APIを10秒ごとに再取得するポーリング方式にする
- 商品在庫状態、ポイント対象商品、商品位置情報を自動更新する

## デザイン
- 緑を基調にする
- 白背景で清潔感を出す
- 黄色はポイント対象商品やお得情報に使う
- スマホ利用を前提にする
- 下部タブでアプリらしく見せる
- 高齢のお客様や忙しい店員でも使いやすいUIにする

## 実装方針
- API通信は services に分ける
- UIは components に分ける
- お気に入りと検索履歴は localStorage を使う
- DB接続設定は PHP 側で config/db.php に分ける
- SQLはプリペアドステートメントを使う
- エラー時もJSONで返す

## 重要
まずはAI機能や自動発注機能は実装しない。
商品検索、売場説明、店内マップ、ポイント対象商品を優先して完成させる。
